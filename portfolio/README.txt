============================================
📝 플로우 상세 페이지 구조
============================================

모든 플로우 페이지는 동일한 구조:
1. 헤더 (프로젝트명, 플로우명)
2. 좌측: 앱/시스템 와이어프레임
3. 우측: 법적 근거 패널
4. 기능: 클릭 시 하이라이트, 위법사항 경고

[법적 근거 데이터 구조]
{
    id: 'law-00',
    title: '법률명 제00조',
    subtitle: '조항명',
    description: '설명',
    checkpoints: ['체크포인트1', '체크포인트2'],
    violationText: '위반 시 경고 메시지'
}


============================================
🔧 새 플로우 추가 방법
============================================

1. 기존 플로우 파일 복사 (예: kt-auth.html)
2. 파일명 변경 (예: kt-transfer.html)
3. 내용 수정:
   - 타이틀 변경
   - screens 배열 수정 (화면 구성)
   - laws 객체 수정 (법적 근거)
4. projects-data.js에서:
   - 해당 플로우의 status를 'ready'로 변경
   - url에 파일 경로 입력


============================================
⚠️ 주의사항
============================================

1. 디자인 일관성 유지
   - 새 페이지 만들 때 기존 스타일 그대로 사용
   - 색상, 폰트, 간격 변경 금지

2. 파일명 규칙
   - 프로젝트ID-플로우ID.html
   - 예: kt-auth.html, lg-iam.html

3. 데이터 수정
   - projects-data.js 파일만 수정
   - HTML 파일 직접 수정하지 않기 (데이터 부분)

4. 이미지/아이콘
   - 현재는 텍스트/이모지 사용
   - 추후 아이콘 추가 시 Lucide 또는 SVG 사용


============================================
📞 작업 요청 시 제공할 정보
============================================

새 플로우 만들 때:
1. 프로젝트명 (K사, LG전자 등)
2. 플로우명 (국외이전, IAM 등)
3. 화면 구성 (어떤 화면들이 있는지)
4. 법적 근거 (해당하는 법 조항들)
5. 검증 체크포인트

예시:
"K사 프로젝트의 '국외이전' 플로우 만들어줘.
화면: 국외이전 동의 → 이전국가 선택 → 수탁자 정보 → 완료
법적 근거: 개인정보보호법 제17조, 제28조의8 등"
